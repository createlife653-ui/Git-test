# コーヒー学習プロセスの可視化

## 目的
- coffee-assetプロジェクトに「学びの物語」を追加する
- 知識の蓄積プロセスを時系列で追えるようにする
- 最終的に「コーヒー学習ガイド」として商品化できる基盤を作る

## 現状の課題
- coffee-assetは知識の結果（原則、パターン、事例）を整理する構造
- 「どのように学んだか」「どのような順番で知識を深めたか」が見えない
- 粗削りな学習プロセスを、他者が役に立つ形に変換する仕組みがない

## 提案する追加ファイル

### 1. LEARNING.md - 学習ジャーナル
学習の物語を時系列で記録するメインファイル

### 2. README.mdへのセクション追加
- 「なぜコーヒーを学び始めたか」
- 「学習のロードマップ」

### 3. 各ファイルへのメタデータ追加
- 作成日、学習の動機、次のステップ

## ファイル構造案

現在構成：
```
coffee-asset/
├── README.md
├── 00_principles/
├── 01_frameworks/
├── 02_case_studies/
└── 03_patterns/
```

追加後：
```
coffee-asset/

├── LEARNING.md（新規：学習ジャーナル）
├── 00_principles/

```
