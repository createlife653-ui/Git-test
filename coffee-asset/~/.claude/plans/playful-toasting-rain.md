# コーヒー栽培地情報記録システム作成計画

## コンテキスト

最初はチェックリスト形式で作成したが、ユーザーからのフィードバックで以下の要件が明らかになった：

1. **チェックボックスではなく、書き込める形式が欲しい**
2. **「自分の気づき」を書く欄を追加したい**
3. **比較できるようにしたい**
4. **国・地域ごとのページを作りたい**

## 新しい設計アプローチ

### 基本コンセプト

国・地域ごとの栽培地情報ファイルを作成し、以下の機能を持たせる：

1. **栽培地情報を記録するスペース**（チェックボックスではなく、直接書き込める形式）
2. **各項目に「自分の気づき」欄を追加**（後から経験と結びつけられる）
3. **同じ国・地域の豆を飲んだ履歴をリンク**（比較しやすくする）
4. **ケーススタディファイルから双方向にリンク**（行ったり来できる）

### template_case.md に追加する内容

```markdown
## 1. Basic Data (Objective Information / 客観情報)
- **Origin (産地)**: Country, Region, Farm
- **Variety (品種)**: e.g., Bourbon, Geisha
- **Process (精製)**: e.g., Washed, Natural
- **Roast Date (焙煎日)**: YYYY/MM/DD
- **Roast Level (焙煎度)**: Light / Medium / Dark

> **Note:** 詳細な栽培地情報を記録・比較したい場合は、[originsディレクトリ](./origins/)で国・地域ごとのファイルに情報を蓄積できます
```

### 使用イメージ

1. 新しい豆（例：インドネシアの豆）を入手したら、`02_case_studies/origins/Indonesia.md` に基本情報を記録
2. ケーススタディファイル（例：`2024-03-09-Indonesia-Ijen.md`）を作成
3. `Indonesia.md` の「飲んだ豆の記録」セクションに、ケーススタディへのリンクを追加
4. 飲んだ後、「自分の気づき」欄に気づいたことを記録
5. 次にインドネシアの豆を飲んだとき、過去の記録と比較できる

### 実行すること

1. `02_case_studies/origins/` ディレクトリを作成
2. 既存のケーススタディから国・地域を抽出し、各ファイルを作成：
   - Indonesia.md
   - Guatemala.md
   - Colombia.md
   - Brazil.md
   - Tanzania.md
   - Honduras.md
3. `01_frameworks/cultivation_checklist.md` を削除
4. `02_case_studies/template_case.md` を更新
