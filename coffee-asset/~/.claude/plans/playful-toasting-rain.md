# コーヒー栽培地情報チェックリスト作成計画

## 目的

コーヒーの勉強のために、栽培地に関する情報を体系的に記録するチェックリストを作成する。

## どこに作るか

**ファイル名:** `cultivation_checklist.md`

**場所:** `01_frameworks/` ディレクトリ

**理由:** このディレクトリは、テスティングフレームワークや評価スケールなど、分析・評価に使うツールを置く場所。

## ファイルの構造

`01_frameworks/_template.md` で決められている構造に従う：

```markdown
# コーヒー栽培地情報チェックリスト

## 何のためにやるの？
コーヒーが育った環境（テロワール）を体系的に記録するため

## 手順
1. 新しい豆を入手したら、パッケージや焙煎店の情報を確認する
2. このチェックリストを使って、入手可能な情報を記録する
3. わかっている範囲で記録する（全ての項目を埋める必要はない）
4. ケーススタディを作成する際に、記録した情報を参照する

## チェックリスト
※チェック項目の内容は自分で調べる

### 1. 地理情報
（項目の内容を自分で調べて記入）

### 2. 気候情報
（項目の内容を自分で調べて記入）

### 3. 土壌・環境情報
（項目の内容を自分で調べて記入）

### 4. 農園情報
（項目の内容を自分で調べて記入）

### 5. その他の情報
（項目の内容を自分で調べて記入）

## うまくいったかな？
実際に使用してからの振り返り用スペース

## 自分との接続
自分の言葉でのまとめ・実際に体験したこと・疑問点を記録するスペース
```

## 大切なルール

**味への影響や用語の解説は書かない**

- ❌ 「高地で育ったコーヒーは酸味が強くなる」
- ❌ 「火山灰土壌はミネラルを豊富に含む」
- ✅ 「標高：1,500-1,800m」
- ✅ 「土壌タイプ：火山灰土」

チェックリストは「事実」だけを記録する。解説や解釈は書かない。

## template_case.md との関連付け

`02_case_studies/template_case.md` の「1. Basic Data」セクションに、以下の追記を追加する：

```markdown
## 1. Basic Data (Objective Information / 客観情報)
- **Origin (産地)**: Country, Region, Farm
- **Variety (品種)**: e.g., Bourbon, Geisha
- **Process (精製)**: e.g., Washed, Natural
- **Roast Date (焙煎日)**: YYYY/MM/DD
- **Roast Level (焙煎度)**: Light / Medium / Dark

> Note: 詳細な栽培地情報（標高・気候・土壌等）が必要な場合は
> [Cultivation Checklist](../01_frameworks/cultivation_checklist.md) を参照して記録
```

## 実行すること

1. `01_frameworks/cultivation_checklist.md` を新規作成する
2. `02_case_studies/template_case.md` に上記の Note を追加する
