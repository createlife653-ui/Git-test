# マルチエージェント・ブレインストーミングシステム実装計画

## コンテキスト

ユーザーはAIを使って「遊びたい」という要望を持っています。具体的には：

- **オートモードでずっと会話を続ける**: 複数のAIが自律的に対話を続けるシステム
- **生産性のある会話**: 単なるおしゃべりではなく、アイデア出し・ブレインストーミングに焦点
- **4人までの役（ペルソナ）**: 異なる視点を持つエージェントが議論
- **無限ループや崩壊対策**: 会話が終わる仕組みが必要
- **新しいアイデアの生成**: 会話から価値あるアイデアを抽出・保存

## 4つのペルソナ設計

### 1. Innovator（イノベーター）
- **役割**: 斬新で革新的なアイデアを提案、既存の枠組みを打破
- **スタイル**: 大胆な表現、「もしこうだったら？」仮説、実験的な考え
- **温度**: 0.9（創造性重視）

### 2. Critic（批評家）
- **役割**: 提案を批判的・論理的に評価、リスクや改善点を指摘
- **スタイル**: 建設的な批判、論理的根拠、改善案の提示
- **温度**: 0.6（論理性重視）

### 3. Synthesizer（統合者）
- **役割**: 複数の意見を統合、新たな視点を提供
- **スタイル**: バランスの取れた表現、妥協点の発見、全体像の俯瞰
- **温度**: 0.7（バランス重視）

### 4. Facilitator（ファシリテーター）
- **役割**: 議論を整理・促進、方向性を示唆
- **スタイル**: 明確で簡潔、要約、次の議論点の提示
- **温度**: 0.5（中立性重視）

## 無限ループ・崩壊対策

### 終了条件（複数組み合わせ）

1. **最大ターン数**: 12ターンで強制終了（設定可能）
2. **類似度チェック**: 直近の発言と80%以上類似している場合に警告
3. **繰り返しパターン検出**: 同じ内容が3回以上繰り返された場合に終了
4. **収束判定**: 直近3ターンの類似度が高い（>0.75）場合に合意形成とみなして終了

### 品質管理

- **短すぎる応答の検出**: 20文字未満は警告
- **有意性チェック**: 具体的なキーワードが含まれているか確認
- **品質スコア**: 各ターンで0.0-1.0のスコアを計算

## アイデア自動抽出

### カテゴリ分類

1. **革新**: 新しいアイデア、提案
2. **改善**: 改善提案、最適化
3. **問題**: 問題提起、課題、リスク
4. **解決策**: 解決方法、対策
5. **アクション**: 次のステップ、実行項目
6. **インサイト**: 気づき、発見

### 抽出フロー

1. 会話ログを文単位に分割
2. 正規表現パターンでカテゴリ分類
3. 確信度スコアを計算（0.5以上のみ採用）
4. 重複除去（類似度85%以上で統合）
5. ソース情報（誰の発言か、何ターン目か）を保持

### 保存形式

- **JSON**: 完全なデータ（会話ログ、メタデータ、アイデア）
- **Markdown**: 読みやすい要約形式（Coffeeプロジェクトのスタイルに合わせる）

## ターン制御アルゴリズム

### Facilitator-Led方式（デフォルト）

1. 3ターンごとにFacilitatorが議論を要約・方向性を提示
2. それ以外は直近の発言者以外からランダムに選択
3. 全エージェントが均等に発言するように配慮

### その他のオプション

- **Round-Robin**: 順番に発言
- **Random**: 完全ランダム
- **Context-Aware**: コンテキストに基づいて選択

## 使用方法

### 基本的な実行

```bash
# Ollamaが起動していることを確認
ollama serve

# プロジェクトディレクトリへ
cd coffee-asset/04_meta/multi_agent_brainstorming

# 依存関係インストール
pip install -r requirements.txt

# 実行
python main.py "新しいコーヒー抽出方法のアイデア"
```

### オプション

```bash
# モデル指定
python main.py "トピック" --model mistral

# ターン数指定
python main.py "トピック" --turns 15

# 出力先指定
python main.py "トピック" --output ../output

# コンソール出力なし（保存のみ）
python main.py "トピック" --quiet
```

### 設定ファイル使用

```bash
python main.py --config my_config.json
```
