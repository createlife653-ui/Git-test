# AI活用度計算スキル作成計画

## Context

コーヒーブログ記事でAIをどの程度活用しているかを可視化するためのスキルを作成します。

記事ごとに「AI活用度」を表示したいというユーザーの要望に基づき、以下の2つの基準でAI活用度を計算します：

- **基準A（文章量ベース）**: AIが生成した文字数の比率
- **基準B（価値ベース）**: AIが代替可能な部分の比率（人間領域を除いたAI生成部分）

## 目的

記事を読み込んで、AI活用度（基準A・基準B）を自動計算するスキルを作成します。

## 実装方針

### 新規作成ファイル

1. `.claude/skills/ai-involvement-calculator/skill.md`
   - スキルのメタデータ定義

2. `.claude/skills/ai-involvement-calculator/prompt.md`
   - AI活用度計算の詳細ロジック

### スキル仕様

**スキル名**: `ai-involvement-calculator`

**トリガー**:
- AI活用度を計算
- 記事のAI活用度
- この記事のAI比率
- ai-involvement-calculator

**パラメータ**:
- `source`: 記事のファイルパス（オプション、指定なしの場合はIDEで開いているファイルを使用）

**出力形式**:
```
## AI活用度計算結果

| 基準 | 値 | 説明 |
|------|-----|------|
| 基準A | XX% | 文章量ベース（AI生成文字数 / 全体文字数） |
| 基準B | XX% | 価値ベース（AI代替可能部分 / 全体文字数） |

### 内訳

- 全体文字数: XXX文字
- 人間領域文字数: XXX文字（詳細：体験XX、感想XX、記録XX）
- AI生成文字数: XXX文字
```

### AI判定ルール（「人間領域」としてカウントするもの）

1. **直接的体験**: 「飲んだ」「味わった」「感じた」など五感を使った表現
2. **個人的感想**: 「おいしい」「面白い」「驚いた」など主観的な評価
3. **具体的記録**: 日付、数値、固有名詞を伴う事実記録
4. **coffee-asset転記**: テイスティングノートからの直接引用
5. **独自の気づき**: ユーザー独自の考察・発見

### 計算式

```
基準A = (AI生成文字数 / 全体文字数) × 100
基準B = ((全体文字数 - 人間領域文字数) / 全体文字数) × 100
```

※ 人間領域には「coffee-assetの記録内容」と「ユーザーが書き足した主観的文章」が含まれる

## 検証方法

1. 実際のブログ記事でスキルを実行
2. 結果の値が直感的に正しいか確認
3. 複数の記事で試して一貫性を確認
