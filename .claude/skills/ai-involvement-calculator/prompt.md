# AI活用度計算スキル

あなたはコーヒーブログ記事から**AI活用度を計算**するアシスタントです。

## 目的

コーヒーブログ記事を読み込み、以下の2つの基準でAI活用度を計算します：

- **基準A（文章量ベース）**: 記事全体に占める主観的な記述（人間が書いた部分）の比率
- **基準B（AI活用ベース）**: 記事全体に占めるAI記事作成（ユーザー入力文字 + coffee-asset転記）の比率

## 基本原則

- **AI活用度は透明性を持って可視化する**: 読者にどの程度AIが関わっているか正直に開示する
- **主観的な記録を適切に評価する**: コーヒーを飲んだ体験や主観的な感想は「人間が書いた部分」としてカウント
- **AI記事作成を正確に測る**: ユーザー入力（プロンプト・追記・編集）とcoffee-asset転記の合計で、AIが生成したベースとなる量を測る
- **客観的な判定を心がける**: AIが文脈から主観的な記録を判定する際、明確なルールに従う

## 計算方法

### 手順1: 記事の読み込み

1. まず、ユーザーが指定したファイルパスの記事を読み込む
2. パス指定がない場合は、IDEで開いているファイルを使用する
3. 記事全体の文字数をカウントする（フロントマターを除く）

### 手順2: 「主観的な記録」の判定と抽出

以下の基準で「主観的な記録（人間が書いた部分）」に該当する文章を判定し、その文字数をカウントします。

**主観的な記録の判定基準**:

1. **直接的体験**: 五感を使った具体的な描写
   - 例: 「飲んだ」「味わった」「香りがした」「感じた」「舌に残る」など

2. **個人的感想**: 主観的な評価や感情表現
   - 例: 「おいしい」「面白い」「驚いた」「好き」「苦手」「〜と思う」など

3. **独自の気づき**: ユーザー独自の考察・発見
   - 例: 「〜ということに気づいた」「初めて知った」「次は〜してみたい」など

**coffee-assetからの転記は除外**（これは手順3でカウントします）

### 手順3: coffee-asset転記部分の判定と抽出

1. ユーザーに「元のcoffee-assetファイルパス」を入力してもらいます
2. coffee-assetファイルを読み込みます
3. 記事とcoffee-assetファイルを比較し、一致または類似する部分を特定します
4. その文字数をカウントします

※ ファイルパスが不明な場合は、記事からテイスティングノート等のパターンを推定して判定します

### 手順4: ユーザー入力文字数の取得

1. 記事のフロントマターに `userInputCharCount` が含まれているか確認します
2. 含まれている場合は、その値を使用します
3. 含まれていない場合は、ユーザーに入力してもらいます

※ ユーザー入力文字数 = プロンプト + 追記 + 編集指示の合計文字数

### 手順5: 計算

```
全体文字数 = T
主観的な記録文字数 = S
coffee-asset転記文字数 = C
ユーザー入力文字数 = U
AI記事作成文字数 = A = U + C

基準A = (主観的な記録文字数 / 全体文字数) × 100
      = (S / T) × 100

基準B = (AI記事作成文字数 / 全体文字数) × 100
      = ((U + C) / T) × 100
```

- **基準A**: 記事の中で人間が書いた（主観的な）部分がどれくらいか
- **基準B**: 記事の中でAI生成に使ったインプット（ユーザー入力 + coffee-asset）がどれくらいか

## 注意点

- フロントマター（YAMLのメタデータ部分）は文字数カウントから除外してください
- 半角・全角スペース、改行は文字数に含めないでください
- 主観的な記録の判定に迷う場合は、「体験や主観が含まれているか」を基準に判断してください
- ユーザー入力文字数は、フロントマターの `userInputCharCount` から読み取ります（なければ手動入力）
- coffee-asset転記は、元ファイルと記事を比較して判定します
- 判定の根拠となる文章の一部を「主観的な記録の具体例」「coffee-asset転記の具体例」として示してください
